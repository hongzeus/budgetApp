# budgetApp AGENTS

## 프로젝트 설명
- 이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
- 주요 대상은 로컬 CSV 파일을 읽고, 거래 내역을 조회/추가/집계/정리하는 Python 명령행 도구다.

## 코딩 규칙
- 모든 함수와 공개 메서드에는 타입 힌트를 반드시 사용한다.
- 함수는 하나당 최대 50줄을 넘기지 않는다.
- 작은 함수로 분해하고, 복잡한 분기는 헬퍼 함수로 분리한다.

## TDD 규칙
- 구현보다 테스트를 먼저 작성한다.
- 새 기능은 실패하는 테스트를 먼저 추가한 뒤 구현한다.
- 버그 수정도 재현 테스트를 먼저 작성한 뒤 수정한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 커지면 함수 분리, 조기 반환, 데이터 구조 단순화를 우선한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 검수 결과에서 테스트 누락, 복잡도 초과, 타입 힌트 누락, 구조적 문제를 확인한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 반드시 커밋하고 푸시한다.
- 기능 단위로 커밋을 쪼개고, 한 커밋에 하나의 목적만 담는다.
